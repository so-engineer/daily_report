## 学習メモ
### Claude Code について
- Claude Code を使えばマネタイズのチャンスがあることが分かった
- エージェントチームの設定方法は、Claude のサイトの設定をコピーしてプロンプトで指示するだけで構築できる

### Rules について
- Rules を使えば CLAUDE.md の内容を分割して、コンテキストを必要なときに動的に使うことができる
- これにより、コンテキストウィンドウの圧縮につなげられるため、大規模なプロジェクトでも効率的に作業できる

### Skills とカスタムスラッシュコマンドについて
- Skills はカスタムスラッシュコマンドが統合された仕組み
- カスタムスラッシュコマンドとは、`.commands` ディレクトリ内の MD ファイルで指示する定型文のこと
- カスタムスラッシュコマンドは Skills 統合後も引き続き使える
- ただし、両方に同じ設定がある場合は Skills の設定が優先される

### Codex と Cline の比較
- Codex はデスクトップアプリのことを指している
- Cline はプロジェクトを精査して全体設計から実装まで一貫して深く行える印象がある
- 一方で Codex は、単発の実装や新機能の追加、バグフィックスといったピンポイントな作業で強みを発揮しそう
- コスト面では、Codex は約 3,000 円のプランで Cline の約 30,000 円プラン相当のトークン量が使えるため、コストパフォーマンスは Codex の方が優れている
