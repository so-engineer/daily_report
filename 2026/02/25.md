## Q. 以下の7つの文章のうち、正しいものを全て選んでください。

1. サービスアカウントのJSONキーは、理由は漏洩リスクが高く、AWSのアクセスキーと同じく管理コストが大きいため、なるべく発行しないほうが良い。
2. Cloud Runでは、サービスアカウントを指定すれば自動的にその権限が付与されるため、GOOGLE_APPLICATION_CREDENTIALSを自分で設定する必要はない。
3. Terraformをローカル環境で実行するには、開発メンバー全員の権限を揃えるためにもサービスアカウントのJSONキーを配布する必要がある。
4. 権限借用（Impersonation）を使えば、JSONキーを使わずに特定のサービスアカウントの権限でTerraformやSDKを実行できる。
5. サービスアカウントは「人間に付与する権限管理の単位」であり、Cloud Runや Cloud SQLなどのサービスに紐づけることは推奨されていない。
6. Cloud Runの実行環境では、自動的に ADC（Application Default Credentials）が有効になり、そこに設定されたサービスアカウントの権限が使用される。
7. オーナーや編集者ロールのような大きすぎるロールをサービスアカウントに付与することは、セキュリティ上避けるべきである。

### A
1, 2, 3, 6, 7
