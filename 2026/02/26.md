## Q1
Cloud SQLにパブリックIPをつけて接続する場合、何が大変ですか？

### A
承認済みネットワーク（許可IP）の管理が大変

## Q2
Cloud SQLに対してプライベートIPで接続している状況下において、Cloud SQL Auth Proxyを導入するメリットを2つ以上挙げてください。

### A
IAMデータベース認証によりユーザー名・パスワードが不要
証明書の管理が不要

## Q3
ローカル環境からCloud SQL Auth Proxy経由でCloud SQLへ接続する場合、

(1) Cloud SQL側でIPの許可リストの管理は (必要/不要)
(2) Cloud SQLにパブリックIPアドレスは (必要/不要)
(3) DBクライアントツールにユーザー名とパスワードの入力は (必要/不要)

それぞれ答えてください。

### A
(1) 不要
(2) 必要
(3) 必要
