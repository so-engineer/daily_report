### Q2. db-migratorをbuildして生成したDockerイメージのタグを、どのようにしてECSタスク定義 db-migrator-stg に渡していますか？
ヒント: GitHub Actions × ECS 自動デプロイの全体像をつかもう！
### A.
パラメータストア経由で渡している。

イメージタグはGHAのbuild時にSSMに渡し、escpressoの定義ファイルから参照している。

### Q3. ecspresso run コマンドは具体的に何をやっていますか？
### A.
ECSタスクの登録・単発実行をしている。

### Q4. ecspresso run コマンドに --watch-container オプションをつけると何が嬉しいですか？ ログ、順番依存 の観点で2つメリットを挙げてください。
### A.
- コンテナのログを標準出力できる
- コンテナの終了をGHA側が待つためマイグレーションが終わる前にワークフローが先に進むといったことが起きず、順番依存を保つことができる
